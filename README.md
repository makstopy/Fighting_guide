# Fighting Guide Expo

## Запуск в режиме разработки (Development Server)

Чтобы запустить сервер разработки (Expo CLI), выполните одну из следующих команд в корневой папке проекта:

```bash
# Использование npm (запустится меню Expo, где можно выбрать эмулятор)
npm start

# Прямой запуск на Android-эмуляторе или подключенном устройстве
npm run android

# Прямой запуск на iOS-симуляторе (требуется macOS)
npm run ios
```

После запуска `npm start` вы можете отсканировать появившийся в терминале QR-код с помощью приложения **Expo Go** на вашем физическом устройстве (Android/iOS) для быстрой проверки приложения.

## Сборка приложения (Production Build)

Для создания установочных файлов (APK/AAB для Android, IPA для iOS) в Expo сейчас стандартным решением является **EAS CLI** (Expo Application Services).

### 1. Установка EAS CLI
Если EAS CLI еще не установлен на вашем компьютере, установите его глобально:
```bash
npm install -g eas-cli
```

### 2. Авторизация
Войдите в свой аккаунт Expo:
```bash
eas login
```

### 3. Запуск облачной сборки
* **Сборка для публикации в Google Play (.aab):**
  ```bash
  eas build -p android --profile production
  ```
  *(По умолчанию этот профиль собирает файл `.aab` (Android App Bundle), который требуется для загрузки в Google Play Store).*

* **Сборка для тестирования на своём устройстве (.apk):**
  ```bash
  eas build -p android --profile preview
  ```
  *(В файле `eas.json` мы явно настроили профиль `preview` на сборку формата `.apk`, чтобы вы могли установить файл напрямую в обход магазина).*

* **Сборка для iOS (.ipa):**
  ```bash
  eas build -p ios
  ```
  *(Потребуется оплаченный аккаунт разработчика Apple).*

### Локальная сборка (на вашем ПК, без облака EAS)
Если вы хотите скомпилировать приложение локально, используйте команды:
```bash
npx expo run:android
npx expo run:ios
```
* **Для Android:** потребуется установленная Android Studio, настроенный Android SDK, NDK и Java.
* **Для iOS:** потребуется компьютер с macOS и установленным Xcode.
